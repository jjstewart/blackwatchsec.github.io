Blog: Dangling DNS – The Invisible Threat That Can Destroy a Business

Most companies are unaware that their unused DNS entries pose a serious security risk. These "dangling DNS" records are subdomains that still resolve to cloud services that have since been decommissioned—leaving an open door for attackers.

Attackers actively scan for such records. Once they identify one, they can spin up a service with the same name in the corresponding cloud provider and hijack the subdomain. This allows them to impersonate the organization, steal data, deliver malware, or host phishing content under a trusted domain.

Real-world examples:

  • Uber had a dangling DNS entry that led to a subdomain takeover exposing sensitive information.
  • Microsoft faced similar issues with Azure-hosted subdomains.

How to prevent it:

  • Regular DNS audits
  • Remove unused DNS entries
  • Monitor cloud asset configurations
  • Use continuous monitoring tools to alert on changes
